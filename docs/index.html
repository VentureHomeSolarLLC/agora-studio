The Problem

Today, Venture Home has three separate knowledge systems that don't talk to each other:

  1. Customer-facing KB (Helios) — Static articles for homeowners
  2. Internal documentation — Process docs, SOPs, tribal knowledge
  3. AI agent skills — What our agents use to answer questions

The result: Customers get different answers depending on who (or what) they ask. When we update one system, the others get stale.

The Solution

A unified knowledge graph where:

  • One source of truth powers customer docs, internal wiki, and AI skills
  • Non-technical users can create content through a visual interface
  • AI automatically classifies content type (concept, lesson, skill)
  • Content syncs everywhere instantly

The Core Insight

Knowledge isn't documents — it's connected concepts.

Traditional: Search KB → Find battery article → Read 3 paragraphs → Find answer Agora: battery-add-on (engram) ├── concept: backup-reserve.md ├── concept: whole-home-vs-partial.md └── lesson: 2026-02-24-customer-expectations.md Agent pulls the exact concept needed, customer sees the full assembled guide.

Architecture

Three-Layer Model

Layer Purpose Examples
Layer 3 Customer-Facing Pages "Battery Backup Guide"
Layer 2 Engrams (Knowledge Units) battery-add-on/, power-outage/
Layer 1 Concept Library backup-reserve.md, net-metering-basics.md

Key Principle: DRY

Instead of copying "How Net Metering Works" into 5 articles, you reference it:

assemble_from:
  - source: concepts/net-metering-basics.md
    sections: [summary, how-credits-work]
  - source: concepts/backup-reserve.md
    sections: all

Update once → Changes appear everywhere.

Visual Page Builder

For Non-Technical Users

  1. Choose a Card (Topic): Battery & Storage, Installation, Post-Install, Billing, etc.
  2. Search & Select Content: Search "backup" → Select storm-guard.md, outage-behavior.md
  3. Arrange & Configure: Drag to reorder, enable TOC, simplify language
  4. Publish: Choose visibility (Customer/Internal/Both), add tags, publish → Live in 2-3 minutes

Behind the Scenes

  1. Visual Builder generates page manifest (YAML)
  2. Backend validates references, generates markdown, commits to GitHub
  3. GitHub Actions rebuilds site, updates Agora index
  4. Live in 2-3 minutes

AI-Powered Content Creation

CS rep writes content → Clicks Analyze → AI classifies:

Detected: CONCEPT (92% confidence)
Title: "Battery Backup Duration"
Tags: [battery, outage, duration]
Related: backup-reserve.md

Auto-generates YAML frontmatter and commits to GitHub.

Human vs. AI Visibility

Every piece of content has an audience field:

  • [customer] — Customer portal only
  • [agent] — AI agents only
  • [customer, agent] — Both (most common)
  • [internal] — Internal team only

Integration with Agora

When you create an Engram with SKILL.md, it becomes a Forge Skill automatically.

Agents query Agora: "Customer says battery isn't backing up their house"
Agora returns: battery-troubleshooting skill + relevant concepts
